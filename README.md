# MLOps MVP Конвейер обработки потоковых данных

Программная реализация системы непрерывного машинного обучения на табличных данных. 
Проект разработан в рамках Задания 1 курса «Интеллектуальный анализ данных (MLOps)». Система эмулирует работу с потоковыми данными, реализуя полный цикл ML-конвейера без использования сторонних фреймворков (MLFlow, Airflow и т.д.), используя только "ванильный" Python и ООП.

## Используемый набор данных
* **Датасет**: Vehicle Insurance Data (motor_data14-2018.csv)
* **Задача**: Мультиклассовая классификация (Колонка `TYPE_VEHICLE`, 11 классов)
* **Временная переменная**: `INSR_BEGIN` (используется для эмуляции хронологического потока данных)

---

## Реализованный функционал

### 1. Сбор данных
* Эмуляция потока: данные считываются батчами по `N` строк (настраивается в конфиге).
* Хранилище сырых данных: батчи сохраняются в файловую систему (`data/raw/`).
* Расчет метапараметров (среднее, дисперсия, кол-во строк/пропусков) с сохранением в JSON.
* **Доп**: Автоматическая генерация случайных пропусков (для демонстрации работы Data Quality).

### 2. Анализ данных
* Оценка Data Quality (поиск пропусков и выбросов по IQR).
* Генерация **ассоциативных правил (Apriori)** на бинаризованных признаках.
* Базовая очистка: автоматическое удаление колонок, если доля пропусков превышает заданный порог.

### 3. Подготовка данных
* Реализован `scikit-learn Pipeline` (`SimpleImputer` -> `StandardScaler` / `OneHotEncoder`).
* Препроцессор обучается на первом батче и замораживается для консистентности признакового пространства.

### 4 & 5. Обучение, дообучение и валидация
* Реализовано Инкрементальное обучение (Incremental Learning).
* Модели: `RandomForestClassifier` и `MLPClassifier`. Обе дообучаются на новых батчах "на лету" (через `warm_start` и `partial_fit`).
* Hold-out валидация на каждом батче. Сохранение метрик (`Accuracy`, `F1-macro`).
* **Хранилище версий (Model Registry)**: система сохраняет артефакты каждой итерации и автоматически определяет лучшую модель за всю историю.

### 6 & 7. Обслуживание и управление
* Управление системой через консольные команды (`argparse`).
* Глобальный конфигурационный файл `config.yaml`.
* Логирование всех процессов (модуль `logging`).

## Структура проекта

```text
├── config.yaml               # Глобальные параметры системы (размеры батчей, гиперпараметры)
├── requirements.txt          # Зависимости Python
├── README.md                 # Документация
├── run.py                    # Контроллер CLI (Точка входа)
├── reset_env.py              # Вспомогательная утилита для сброса состояния
│
├── data/                     # Хранилище данных
│   ├── initial/              # Исходный CSV датасет (motor_data14-2018.csv)
│   ├── raw/                  # Сырые батчи (эмуляция потока)
│   ├── processed/            # Очищенные батчи после этапа Data Quality
│   └── external/             # Данные и результаты для инференса
│
├── storage/                  # Файловая БД системы
│   ├── metrics/              # JSON-отчеты (Метапараметры, DQ, Метрики ML, Правила Apriori)
│   ├── models/               # Реестр моделей (.pkl файлы и model_registry.json)
│   ├── reports/              # Итоговые Markdown-отчеты (Summary) и системные логи
│   └── stream_state.json     # Файл состояния "курсора" потока данных
│
└── src/                      # Исходный код (Модели и логика)
    └── models/
        ├── data_handler.py   # Сбор и эмуляция потока
        ├── data_analyzer.py  # Data Quality и Apriori
        └── ml_models.py      # ML pipeline, обучение и версионирование
```

## Установка и запуск

1. Создайте виртуальное окружение и активируйте его:
   ```bash
   python -m venv .venv
   source .venv/bin/activate
   ```
2. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
3. Поместите исходный датасет `motor_data14-2018.csv` в директорию `data/initial/` (и так есть в гите).

## Сценарии использования (CLI Интерфейс)

Система управляется через главный скрипт `run.py`.

### Вспомогательная команда (Сброс)
Если необходимо начать эксперимент с нуля, очистить историю моделей и сбросить курсор потока, выполните:
```bash
python reset_env.py
```

### 1. ыПолучение нового батча и дообучение (Update)
Эмулирует приход новой порции данных. Выполняет весь цикл: Сбор -> DQ -> Apriori -> Препроцессинг -> Дообучение -> Оценка -> Сохранение в Registry.
```bash
python run.py -mode update
```
*(Эту команду стоит запустить несколько раз подряд для наглядности)*

### 2. Применение лучшей модели к новым данным (Inference)
Система автоматически обращается к `model_registry.json`, находит лучшую модель (Champion) по историческим метрикам, загружает её и строит предсказания для переданного файла.
```bash
python run.py -mode inference -file data/raw/batch_1.csv
```
*Результат будет сохранен в `data/external/predictions_output.csv` с добавленной колонкой `predict`.*

### 3. Формирование отчета (Summary)
Генерирует агрегированный Markdown-отчет об изменении качества работы моделей во времени (по всем обработанным батчам).
```bash
python run.py -mode summary
```
*Отчет сохраняется в `storage/reports/summary_report.md`.*

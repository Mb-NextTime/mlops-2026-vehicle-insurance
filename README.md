# MLOps MVP Конвейер обработки потоковых данных

Программная реализация системы непрерывного машинного обучения на табличных данных. 
Проект разработан в рамках задания курса по MLOps. Система эмулирует работу с потоковыми данными, реализуя полный цикл ML-конвейера без использования сторонних фреймворков (MLFlow, Airflow и т.д.).

## Сценарии использования (CLI Интерфейс)

Система управляется через главный скрипт `run.py`. Перед началом работы установите зависимости `pip install -r requirements.txt`.

*   **Очистка системы (сброс к нулю):** `python reset_env.py`
*   **Итерация потока (получение батча + дообучение):** `python run.py -mode update`
*   **Генерация отчета:** `python run.py -mode summary`
*   **Инференс лучшей модели:** `python run.py -mode inference -file data/raw/batch_1.csv`

## CI/CD Интеграция (GitHub Actions)

В проекте реализован CI/CD workflow через CRON-инкрементальное обучение, настроенный в `.github/workflows/mlops_pipeline.yml`.

### Инструкция по развертыванию
1. Сделайте Fork или загрузите проект в собственный репозиторий на GitHub.
2. Убедитесь, что исходный датасет `data/initial/motor_data14-2018.csv` находится в репозитории.
3. Перейдите во вкладку **Actions** в вашем репозитории и включите выполнение workflows.
4. Конвейер будет запускаться автоматически при коммитах в ветку `main`, по Pull Request-ам, а также **ежедневно по расписанию в 00:00 (cron)**.
5. Доступен ручной запуск через кнопку **Run workflow** (workflow_dispatch).

### Доступные артефакты
После каждого запуска CI/CD (во вкладке Actions -> Run) формируется архив `mlops-artifacts`, содержащий:
* Системные логи обучения (`system.log`).
* Лучшие сериализованные модели (`.pkl` и `model_registry.json`).
* Файл состояния сборщика данных (`stream_state.json`).
Также во вкладке **Summary** конкретного запуска генерируется Markdown-дашборд с историей изменения качества моделей.
